# IOST Web Wallet Application (PWA) - spec.md

## 1. プロジェクト概要
IOSTレイヤー1ブロックチェーンを利用した、モバイルファーストのWebアプリケーション（PWA）。
複雑なブロックチェーンの概念を隠蔽し、一般的な決済アプリ（PayPay等）のような直感的でユーザーフレンドリーなUI/UXを提供する。ユーザーは一度IOSTの秘密鍵をインポートしてGoogleアカウントと紐付けることで、以降はGoogleログインのみで残高確認やステーキングなどの操作が可能になる。

## 2. システムアーキテクチャ
* **フロントエンド:** React (Next.js または Vite), TypeScript
* **スタイリング:** Tailwind CSS (添付画像のようなクリーンなコンポーネント実装のため)
* **バックエンド / 認証:** Firebase Authentication (Google OAuth), Cloud Firestore
* **ブロックチェーン連携:** IOST Javascript SDK (`iost.js`)
* **ホスティング:** Vercel または Firebase Hosting

## 3. セキュリティ・秘密鍵管理要件（最重要）
* **非カストディアル（自己管理）原則:** サーバー（Firestore）に秘密鍵を平文で保存しない。
* **暗号化プロセス:** 1. ユーザーが「IOSTアカウント名」と「秘密鍵」を入力。
    2. クライアント側（ブラウザ内）で、ユーザー固有のPINコードまたはGoogle Authのトークンを元に鍵を生成し、IOSTの秘密鍵をAES暗号化。
    3. 暗号化された秘密鍵（Encrypted Keystore）のみをFirestoreに保存。
    4. トランザクション実行時（ステーキングや送金）は、セッション内で復号化し `iost.js` の `KeyPair` インスタンスを生成して署名を行う。

## 4. 画面・UI仕様 (UI/UX Design)
添付の決済アプリデザインをベースにしたモバイルファースト設計。

### 4.1. 共通コンポーネント
* **Bottom Navigation Bar:** 「ホーム」「ステーキング」「履歴」「設定」の4タブ。
* **テーマカラー:** プライマリーカラー（例: ブランドレッド `#E60012`）、背景（ライトグレー `#F8F9FA`）、カード（白 `#FFFFFF`）。

### 4.2. 各画面の仕様
1.  **ログイン / オンボーディング画面**
    * 「Googleでログイン」ボタン。
    * 初回ログイン時: 「IOSTアカウント名」と「秘密鍵」のインポートフォーム画面への遷移。
    * 暗号化用PINコードの設定（オプションだが推奨）。

2.  **ホーム画面 (ダッシュボード)**
    * **ヘッダー:** ユーザーのアカウント名表示。
    * **残高表示エリア:** 大きなフォントでIOST残高を表示（例: `20,250 IOST`）。日本円（JPY）の換算レートも小さく併記。
    * **クイックアクション（丸型ボタン）:**
        * 「送る (Send)」
        * 「受け取る (Receive / QRコード表示)」
        * 「増やす (Stake)」
    * **最近のトランザクション:** 下部に数件の履歴をリスト表示。

3.  **ステーキング画面 (Staking)**
    * IOSTの「Gas/RAMリソース取得」または「ノードへの投票（Vote）」を行う画面。
    * 現在のステーキング量と獲得報酬の表示。
    * 「ステーキングする」「解除する」ボタンと、数量入力フィールド。

## 5. 機能要件詳細
* **Google OAuth連携:** Firebase Authを使用したログイン・セッション管理。
* **IOST残高取得:** IOST RPC API (`getAccount` エンドポイント) を使用し、アカウント残高をリアルタイム取得。
* **IOSTトランザクション作成:**
    * `iost.js` を使用した `iost.createTx()` および `iost.signAndSend()` の実装。
    * 送金: `token.iost` コントラクトの `transfer` 呼び出し。
    * ステーキング: `gas.iost` (Gas取得) または `vote_producer.iost` (ノード投票) コントラクトの呼び出し。
* **PWA対応:** `manifest.json` と Service Worker の設定。スマホのホーム画面に追加してネイティブアプリのように起動可能にする。

## 6. AI開発エージェントへの指示（実装ステップ）
1.  **フェーズ1 (Setup):** Next.js、Tailwind CSS、Firebaseの初期セットアップを行い、モバイルレイアウトのモックを作成してください。
2.  **フェーズ2 (Auth & Security):** Googleログイン機能と、秘密鍵をクライアント側で暗号化してFirestoreに保存・取得するロジックを実装してください。
3.  **フェーズ3 (IOST Integration):** `iost.js` をインストールし、IOSTテストネット（またはメインネット）のRPCノードに接続して残高を表示するカスタムフックを作成してください。
4.  **フェーズ4 (Staking & Tx):** 実際にIOSTをステーキング（または送金）するトランザクションを構築し、暗号化された鍵を復号して署名・送信する機能を追加してください。
