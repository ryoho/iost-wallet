# IOST Web Wallet Application (PWA) - spec-phase2.md

## 2. ディレクトリ・コンテキスト
* **現在のプロジェクト:** カレントディレクトリ (`/home/rio/VSC/iost`)

## 3. 機能要件：IOSTアカウント無料作成機能（代払い実装）
ユーザーが初期費用（RAM/Gas）なしでIOSTアカウントを作成できる機能を追加する。
作成にかかるコストは、事前に資金がデポジットされたスポンサーアカウント `aiyaa_cs` が負担する。

### 3.1. セキュリティ要件（最重要）
* **秘密鍵の保護:** `aiyaa_cs` の秘密鍵は、絶対にフロントエンド（クライアント側）のコードに含めないこと。
* **バックエンド処理:** Next.js の API Routes (例: `app/api/create-account/route.ts`) または Server Actions を作成し、サーバーサイドで `aiyaa_cs` を用いた署名とトランザクション送信を行うこと。

### 3.2. 処理フロー
1. **フロントエンド:** ユーザーが希望する「新しいIOSTアカウント名」を入力し、バックエンドAPIにリクエストを送信。
2. **バックエンド (API):**
   * `iost.js` を使用し、新しいキーペア（公開鍵・秘密鍵）を生成。
   * スポンサーアカウント (`aiyaa_cs`) を用いて、ネットワークに `auth.iost` (アカウント作成) と `gas.iost` / `ram.iost` (リソース購入) のトランザクションを送信・署名する。
3. **フロントエンドへの返却:** アカウント作成成功後、生成された「新しい秘密鍵」を暗号化してFirestore等に保存するプロセス（フェーズ2の認証フロー）へと繋ぐ。

## 4. UI/UXとアセットのマイグレーション
旧プロジェクトのデザイン要素（キャラクター、特定の画像素材など）を踏襲しつつ、フェーズ1で作成したクリーンなUIに合わせてモダンにアップデートする。

### 4.1. アセットの移行
* `/home/rio/VSC/bk_my-app` 内（主に `public` や `src/assets` フォルダ内）に存在する画像・キャラクター素材（`.png`, `.jpg`, `.svg` 等）を抽出し、現在のプロジェクトの `public` フォルダへコピーする。

### 4.2. デザインのアップデート
* コピーした画像やキャラクターを、ログイン画面やホーム画面の適切な場所に配置する。
* Tailwind CSSを活用し、古いデザインをそのまま再現するのではなく、カードデザイン、シャドウ、角丸（rounded）、適切な余白（padding/margin）を用いて「モダンなスマホ決済アプリ風」に昇華させること。
