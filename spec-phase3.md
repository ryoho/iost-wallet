# IOST Web Wallet Application (PWA) - spec-phase3.md

## 1. フェーズ3 プロジェクト概要
アプリケーションのUI/UXを大幅にブラッシュアップし、ユーザーフレンドリーで親しみやすいデザインへと改修する。また、スマートフォンでの実機動作確認のため、Web上（Vercelを想定）へのデプロイ準備を完了させる。

## 2. UI/UX 改修要件 (Friendly & Soft Design)
現在の「原色寄り・余白なし」のデザインを破棄し、以下のガイドラインに従ってTailwind CSSのクラスを全面的に見直す。

### 2.1. カラースキーム（パステル・中間色）
* **背景色:** 画面全体の背景は真っ白ではなく、極めて薄いグレーやオフホワイト（例: `bg-slate-50` や `bg-gray-50`）を使用し、目に優しくする。
* **カード・コンポーネント:** 白（`bg-white`）を使用し、背景とのコントラストで浮き立たせる。影は柔らかく大きく（例: `shadow-sm` や `shadow-md`）。
* **アクセントカラー（ボタン等）:** 原色（真っ赤や真っ青）は禁止。パステル調または少し彩度を落とした中間色（例: `bg-rose-400`, `bg-sky-400`, `bg-teal-400` など）を使用する。
* **テキスト:** 真っ黒（`#000000`）は避け、濃いグレー（例: `text-slate-700` や `text-gray-600`）を使用する。

### 2.2. 余白（ホワイトスペース）と角丸
* **Padding / Margin:** コンテンツが画面端や要素同士でくっつかないよう、十分な余白（`p-4`, `p-6`, `gap-4`, `gap-6` 等）を確保する。
* **角丸 (Border Radius):** ボタンやカードコンポーネントは、丸みを帯びたフレンドリーな形状（`rounded-xl`, `rounded-2xl`, または `rounded-full`）を徹底する。

### 2.3. 絵文字の活用
* 堅苦しいテキストラベルだけでなく、絵文字をアイコン代わりに併記して直感的にする。
  * 例: "送る" -> "💸 送る" または "📤 送金"
  * 例: "受け取る" -> "📥 受け取る"
  * 例: "ステーキング" -> "🥩 ステーキング" または "🌱 増やす"
  * 例: "ホーム" -> "🏠 ホーム"

## 3. デプロイ要件 (Deployment Preparation)
Next.jsアプリケーションをVercelへデプロイするための準備を行う。

* **ビルドチェック:** `npm run build` がエラーなく通る状態にする（TypeScriptの型エラーや、未使用変数の警告などがあれば修正する）。
* **環境変数:** デプロイ先で必要になる環境変数（Firebaseの設定や `aiyaa_cs` の秘密鍵など）を整理し、`.env.example` ファイルとしてリストアップする（実際の値は含めない）。

## 4. AIエージェントへの指示（実装ステップ）
1. **ステップ1 (UI Refactoring):** 全てのページ（ログイン画面、ホーム画面など）とコンポーネントのTailwindクラスを見直し、上記の「パステルカラー」「十分な余白」「大きな角丸」「絵文字」を適用したフレンドリーなデザインに書き換えてください。
2. **ステップ2 (Build Fixes):** `npm run build` を実行し、もしエラーが出た場合は全て修正して、本番環境へデプロイ可能な状態にしてください。
3. **ステップ3 (Env Setup):** 現在使用している環境変数をまとめた `.env.example` ファイルを作成してください。
