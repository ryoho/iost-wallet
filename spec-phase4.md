# IOST Web Wallet Application (PWA) - spec-phase4.md

## 1. フェーズ4 プロジェクト概要
本格的なウォレットとしての利便性を高めるため、外部APIを利用した「IOST/JPYのリアルタイム価格取得と評価額表示」、およびFirestoreを利用した「単一Googleアカウントでの複数IOSTアカウント管理機能」を実装する。また、それに伴うログイン直後のルーティング（オンボーディングフロー）を最適化する。

## 2. 新規機能要件

### 2.1. IOST最新価格（日本円）の取得と表示
* **API連携:** CoinGecko API（`https://api.coingecko.com/api/v3/simple/price?ids=iostoken&vs_currencies=jpy`）またはBinance等のパブリックAPIを利用し、IOSTの現在の日本円（JPY）価格を取得する。
* **UI表示:** * ホーム画面（ダッシュボード）の残高表示部分をアップデートする。
  * 現在の保有IOST枚数の下に、日本円換算の総評価額（例: `≈ ¥15,430`）を大きく表示する。
  * 画面のどこかに現在のレート（例: `1 IOST = ¥1.24`）を小さく併記する。

### 2.2. 複数アカウント管理（マルチアカウント）とデータ構造
* **Firestoreデータ構造の変更:** ユーザー（Google UID）に対して、複数のIOSTアカウント（暗号化された秘密鍵のセット）を保存できるように配列またはサブコレクションの形式にアップデートする。
* **アカウント切り替えUI:** ヘッダー部分などに「現在のアクティブアカウント」を表示し、タップするとドロップダウンまたはボトムシートが開き、別のアカウントへの切り替え、追加、削除が行えるようにする。

### 2.3. アカウント追加フローの分岐
新しいアカウントを追加する際、以下の2つの選択肢をユーザーに提示するUI（カード選択式など）を作成する。
1. **「✨ 新しく作成する (無料)」:** フェーズ2で実装した `aiyaa_cs` アカウントによる代払いAPI（新規作成ロジック）を呼び出す。
2. **「🔑 既存のアカウントをインポート」:** IOSTアカウント名と秘密鍵を入力し、暗号化して保存する従来のフロー。

### 2.4. スマート・オンボーディング（ログインフローの改善）
* ユーザーが「Googleでログイン」した直後のルーティングを以下のロジックに改修する。
  1. Firestoreを参照し、紐づいているIOSTアカウントが存在するか確認する。
  2. **アカウントが0件の場合:** 強制的に「アカウント追加画面（2.3の選択肢画面）」へリダイレクトする。
  3. **アカウントが1件以上ある場合:** ホーム画面へ遷移し、前回アクティブだった（またはリストの先頭の）アカウントの情報を表示する。

## 3. AIエージェントへの指示（実装ステップ）
1. **ステップ1 (Database & Auth Routing):** Firestoreのデータ取得ロジックを見直し、複数アカウントを配列等で管理できるように型定義と関数を修正してください。その後、ログイン直後にアカウントが0件なら `/onboarding`（または該当の画面）へ、存在すれば `/dashboard` へ遷移するルーティングを実装してください。
2. **ステップ2 (Onboarding UI):** `/onboarding` 画面を実装し、「新規作成（代払いAPIをコール）」と「既存インポート」の2つのカードボタンから選べる分かりやすいUIを作成してください。
3. **ステップ3 (Multi-Account Switcher):** ホーム画面のヘッダー等に、現在のアカウント名を表示し、クリックで他のアカウントへの切り替え・追加・削除ができるメニュー（ドロップダウン等）を実装してください。
4. **ステップ4 (Price Fetching):** SWRやReact Query、またはカスタムフックを使用してCoinGecko等のAPIからIOSTのJPY価格を取得し、ホーム画面の残高を「日本円の評価額」と共にリッチに表示するようにUIをアップデートしてください。
