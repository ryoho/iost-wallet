# IOST Web Wallet Application (PWA) - spec-phase5.md

## 1. フェーズ5 プロジェクト概要
プロトタイプ段階のコードベースを整理（リファクタリング）し、メンテナンス性を向上させる。また、ローカルに用意されたHTMLテンプレートをベースにしたミニマルなデザインへの刷新、Google OAuthを起点とした厳密なルーティングの構築、およびIOSTアカウントのトランザクション履歴取得機能を実装する。

## 2. 機能要件詳細

### 2.1. デザインの刷新 (UI/UX Redesign)
* **テンプレートの適用:** カレントディレクトリに存在する `/home/rio/VSC/iost/design-template-01.html` の構造とCSS（Tailwind）を読み解き、Reactコンポーネントに移植する。
* **アセットの断捨離:** これまで使用していた静的な画像ファイル（旧アプリから移行したキャラクター画像など）は `public` フォルダから全て削除する。
* **ミニマリズム:** 画像に頼らず、絵文字（🌸, 💸, 🥩など）や、軽量なSVGアイコンライブラリ（`lucide-react` 等）のみを使用したクリーンでシンプルなデザインを構築する。
* **ティッカーシンボルの動的取得:** IOSTのロゴやティッカーアイコンは、静的ファイルとして持たず、CoinGecko APIなどの外部ソース、またはIOST公式の静的アセットURLから動的に取得して表示する。

### 2.2. コードの最適化とリファクタリング (Refactoring)
* これまでのフェーズで蓄積した技術的負債を解消する。
* 重複している処理（特にFirebase Firestoreへの読み書きロジックや、IOST APIの呼び出し部分）を、カスタムフック（例: `useFirestore`, `useIOST`）やユーティリティ関数として分離する。
* 不要な `console.log` や、使用されていない変数・コンポーネントを削除し、DRY（Don't Repeat Yourself）原則を徹底する。

### 2.3. ルーティングとオンボーディングフローの厳密化
* 認証状態（Auth State）に基づいた厳密なルート保護（Route Guards）を実装する。
* **初期画面 (`/` または `/login`):** 一切の無駄を省き、「Googleでログイン」ボタンのみを配置したシンプルな画面にする。
* **ログイン後の分岐ロジック:**
  1. Google認証完了後、Firestoreを参照し、紐づくIOSTアカウントの配列を確認する。
  2. **紐づくアカウントが0件の場合:** 自動的に `/onboarding` 画面へ遷移させる。ここでは「新規作成（代払い）」または「秘密鍵のインポート」のみを選択させる。
  3. **紐づくアカウントが1件以上ある場合:** 自動的に `/dashboard`（ホーム画面）へ遷移させる。

### 2.4. トランザクション履歴の取得 (Tx History)
* インポート（または作成）したIOSTアカウントの過去のトランザクション履歴を取得し、UIにリスト表示する機能を実装する。
* **実装アプローチ:**
  1. まずは `iost.js` （公式SDK）や、IOSTのネイティブRPC APIを利用して履歴の取得を試みる。
  2. もしネイティブAPIでの履歴取得が困難（または十分なデータが得られない）場合は、ブロックチェーンエクスプローラー（例: IOSTABC）が提供するパブリックAPI（REST API）経由での取得ロジックに切り替えて実装する。

## 3. AIエージェントへの指示（実装ステップ）
1. **ステップ1 (Cleanup & Refactor):** 既存のコードベースを解析し、使用されていない画像アセットの削除、および冗長なロジックのカスタムフック化・リファクタリングを実行してください。
2. **ステップ2 (Strict Routing):** 仕様 2.3 に従い、初期画面を「Googleログインのみ」に変更し、認証成功後にFirestoreのデータ有無に応じて `/onboarding` または `/dashboard` に正しく振り分けるルーティングフローを構築してください。
3. **ステップ3 (Template Integration):** `/home/rio/VSC/iost/design-template-01.html` を読み込み、そのデザインとレイアウト構造を抽出して、Next.jsの各画面（ダッシュボード等）に適用してください。画像は使わず、絵文字や `lucide-react` を利用してください。
4. **ステップ4 (Tx History):** ダッシュボードまたは履歴タブにおいて、対象のIOSTアカウントの過去のトランザクション履歴を取得・表示する機能を実装してください。APIの制約に注意し、最適な取得方法を選択してください。
